üîó API Endpoints Reference

Real-Time Deposit Endpoints

POST /api/deposits/realtime/start

Start real-time deposit monitoring for immediate detection

Example: curl -X POST http://localhost:5001/api/deposits/realtime/start
POST /api/deposits/realtime/stop

Stop real-time deposit monitoring

Example: curl -X POST http://localhost:5001/api/deposits/realtime/stop
GET /api/deposits/realtime/next?timeout=30

üéØ KEY ENDPOINT: Wait for next deposit (blocks until deposit happens) - Returns: wallet_address, hash, timestamp, amount

Example: curl "http://localhost:5001/api/deposits/realtime/next?timeout=30"
GET /api/deposits/realtime/stream

Server-Sent Events stream for continuous real-time updates

JavaScript: new EventSource('/api/deposits/realtime/stream')
GET /api/deposits/realtime/status

Get real-time monitor status and statistics

Example: curl http://localhost:5001/api/deposits/realtime/status

Standard Endpoints

GET /api/health

Health check and system status

Example: curl http://localhost:5001/api/health
GET /api/transactions

Get transactions with filters (limit, min_amount, sender_address, etc.)

Example: curl "http://localhost:5001/api/transactions?limit=10&min_amount=0.1"
GET /api/deposits/recent

Get recent deposits in JSON format (historical data)

Example: curl "http://localhost:5001/api/deposits/recent?limit=10"
GET /api/balance

Get current wallet balance

Example: curl http://localhost:5001/api/balance
POST /api/sync

Manually sync transactions from blockchain

Example: curl -X POST http://localhost:5001/api/sync
